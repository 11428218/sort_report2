## 氣泡排序法（Bubble Sort）

### 原理簡述

氣泡排序法會重複走訪陣列，依序比較相鄰兩個元素，若順序錯誤就交換。每完成一輪，當前最大值會被推到右側正確位置。

### 流程重點

1. 從索引 0 開始比較相鄰元素。
2. 若左值大於右值則交換。
3. 一輪結束後，最右側元素固定。
4. 若某輪沒有交換，可提前結束。

### 複雜度分析

- 最佳時間複雜度：O(n)
- 平均時間複雜度：O(n^2)
- 最差時間複雜度：O(n^2)
- 空間複雜度：O(1)
- 穩定性：穩定排序

### 模擬實作內容

Bubble Sort 頁面提供輸入陣列、隨機產生、速度控制、開始排序、停止與單步執行等功能，能清楚觀察每一次比較與交換。

## 選擇排序法（Selection Sort）

### 原理簡述

選擇排序每一輪從未排序區間中找出最小值，與當前輪起始位置交換，使左側區間逐步成為已排序狀態。

### 流程重點

1. 第 i 輪先假設第 i 個元素是最小值。
2. 掃描 i 到 n-1，找出真正最小值索引。
3. 將最小值與 i 位置交換。
4. i 往右推進，直到排序完成。

### 複雜度分析

- 最佳時間複雜度：O(n^2)
- 平均時間複雜度：O(n^2)
- 最差時間複雜度：O(n^2)
- 空間複雜度：O(1)
- 穩定性：不穩定排序

### 模擬實作內容

Selection Sort 頁面可即時顯示當前輪次、比較次數、交換次數與目前最小值位置，適合觀察每輪只做一次交換的特性。

## 插入排序法（Insertion Sort）

### 原理簡述

插入排序將陣列分成已排序區與未排序區，每次取未排序區第一個元素，插入到已排序區中的正確位置。

### 流程重點

1. 將第一個元素視為已排序。
2. 從第二個元素開始，記錄當前值 key。
3. 向左比較已排序區元素，較大者右移。
4. 找到位置後放入 key。

### 複雜度分析

- 最佳時間複雜度：O(n)
- 平均時間複雜度：O(n^2)
- 最差時間複雜度：O(n^2)
- 空間複雜度：O(1)
- 穩定性：穩定排序

### 模擬實作內容

Insertion Sort 頁面除了基本的開始、暫停、單步與重置外，也能調整資料筆數與速度。畫面會同步顯示比較次數、移動次數與目前狀態。

## 合併排序法（Merge Sort）

### 原理簡述

合併排序法採用分治法（Divide and Conquer），先把陣列遞迴切分成小區段，再將已排序的小區段逐步合併成大區段。

### 流程重點

1. 將陣列切成左右兩半，直到子陣列長度為 1。
2. 分別對左右子陣列排序。
3. 比較左右子陣列元素，依序合併為有序區段。
4. 重複合併直到整體完成排序。

### 複雜度分析

- 最佳時間複雜度：O(n log n)
- 平均時間複雜度：O(n log n)
- 最差時間複雜度：O(n log n)
- 空間複雜度：O(n)
- 穩定性：穩定排序

### 模擬實作內容

Merge Sort 頁面可逐步觀察區段合併、比較與寫入過程，並同步顯示合併輪次、比較次數與寫入次數。

## 雞尾酒排序法（Cocktail Sort）

### 原理簡述

雞尾酒排序法是雙向版的氣泡排序。每一輪會先由左到右把大值推到右端，再由右到左把小值推到左端。

### 流程重點

1. 正向掃描並交換逆序的相鄰元素。
2. 正向掃描後縮小右邊界。
3. 反向掃描並交換逆序的相鄰元素。
4. 反向掃描後縮小左邊界，重複直到無交換。

### 複雜度分析

- 最佳時間複雜度：O(n)
- 平均時間複雜度：O(n^2)
- 最差時間複雜度：O(n^2)
- 空間複雜度：O(1)
- 穩定性：穩定排序

### 模擬實作內容

Cocktail Sort 頁面可視覺化正向與反向兩段掃描，並顯示每輪比較與交換統計，適合對照 Bubble Sort 的差異。

## 綜合比較

| 演算法 | 最佳時間 | 平均時間 | 最差時間 | 空間 | 穩定性 | 特性摘要 |
|---|---|---|---|---|---|---|
| Bubble Sort | O(n) | O(n^2) | O(n^2) | O(1) | 穩定 | 實作簡單，適合教學 |
| Selection Sort | O(n^2) | O(n^2) | O(n^2) | O(1) | 不穩定 | 比較次數固定，交換次數少 |
| Insertion Sort | O(n) | O(n^2) | O(n^2) | O(1) | 穩定 | 近乎有序資料表現佳 |
| Merge Sort | O(n log n) | O(n log n) | O(n log n) | O(n) | 穩定 | 分治法，資料量大時更有效率 |
| Cocktail Sort | O(n) | O(n^2) | O(n^2) | O(1) | 穩定 | 雙向掃描，改善部分逆序情況 |

### 使用建議

- 小型或近乎有序資料：Insertion Sort
- 教學與基礎理解：Bubble Sort、Selection Sort
- 資料量較大且重視時間效率：Merge Sort
- 需要雙向交換觀察流程：Cocktail Sort

## 複雜度實驗（以氣泡排序為例）

以下用和課堂範例同樣的方式呈現複雜度實驗，內容分成三個部分。

### 1. `bubble_sort` 函式說明

- 外迴圈 `k` 控制當輪比較範圍，會從 `n-1` 逐輪遞減。
- 內迴圈 `i` 執行相鄰比較，若 `a[i] > a[i+1]` 則交換。
- 交換採三步驟與暫存變數 `t`，避免直接覆蓋造成原值遺失。

對應程式檔案：[complexity_bubble_experiment.c](complexity_bubble_experiment.c)

### 2. `main` 函式與計時方式

- 初始化「完全逆序」陣列，作為氣泡排序最差情況。
- 使用 `clock()` 在排序前後取樣 CPU ticks。
- 用 `(end - start) / CLOCKS_PER_SEC` 轉換為秒數，便於比較不同 `N`。

### 3. 實驗表格與 O(n²) 驗證

建議測試 `N`：1000、2000、5000、10000、20000、50000、100000。

可在報告中填寫下表：

| N | 時間 (秒) | N 增加倍數 | 時間增加倍數 |
|---:|---:|---:|---:|
| 1,000 |  | -- | -- |
| 10,000 |  | x10 | 預期約 x100 |
| 100,000 |  | x100 | 預期約 x10000 |

判讀方式：

- 若 `N` 增加 10 倍時，時間約增加 100 倍，則與 O(n²) 結論一致。
- 若偏差較大，通常與硬體、編譯器最佳化、背景程式負載有關。
